# 多轮对话设计与实现

## 概述

本项目基于 Spring AI 的 **ChatMemory + Advisor 模式**实现多轮对话，底层存储使用 MySQL，具备持久化、多租户隔离、上下文管理等特点。

## 数据模型

### 数据库表结构

```sql
CREATE TABLE conversation (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id VARCHAR(64) NOT NULL,           -- 租户ID（隔离不同客户的数据）
    conversation_id VARCHAR(255) NOT NULL,    -- 会话ID，格式: "type::chatId"
    message TEXT NOT NULL,                    -- 消息内容
    type VARCHAR(32) NOT NULL,                -- 角色: USER / ASSISTANT / SYSTEM
    create_time DATETIME NOT NULL,            -- 创建时间
    INDEX idx_tenant_conversation (tenant_id, conversation_id),
    INDEX idx_create_time (create_time)
);
```

## ConversationId 设计

### 格式规范

```
conversationId = type + "::" + chatId
```

### 示例

| type | chatId | conversationId |
|------|--------|----------------|
| chat | abc123 | `chat::abc123` |
| service | xyz789 | `service::xyz789` |
| pdf | doc456 | `pdf::doc456` |
| react | agent001 | `react::agent001` |

### type 业务场景

| type | 说明 |
|------|------|
| chat | 通用聊天 |
| service | 客服问答（支持工具调用） |
| pdf | PDF 知识库问答（RAG 增强） |
| react | ReAct 智能体 |

### 工具类实现

```java
public final class ConversationIdHelper {
    private static final String SEPARATOR = "::";

    public static String build(String type, String chatId) {
        if (!StringUtils.hasText(type) || !StringUtils.hasText(chatId)) {
            throw new IllegalArgumentException("type and chatId must not be blank");
        }
        return type + SEPARATOR + chatId;
    }

    public static String extractChatId(String conversationId) {
        if (!StringUtils.hasText(conversationId)) {
            return conversationId;
        }
        int index = conversationId.indexOf(SEPARATOR);
        return index < 0 ? conversationId : conversationId.substring(index + SEPARATOR.length());
    }
}
```

## ChatMemory 实现

### MysqlChatMemory 核心逻辑

```java
@Component
public class MysqlChatMemory implements ChatMemory {
    private final ConversationMapper conversationMapper;

    // 添加消息到数据库
    @Override
    public void add(String conversationId, List<Message> messages) {
        String tenantId = currentTenantId();
        for (Message message : messages) {
            Conversation conv = Conversation.builder()
                .tenantId(tenantId)
                .conversationId(conversationId)
                .message(message.getText())
                .type(message.getMessageType().getValue())  // USER / ASSISTANT / SYSTEM
                .createTime(LocalDateTime.now())
                .build();
            conversationMapper.insert(conv);
        }
    }

    // 获取最近 N 条消息
    @Override
    public List<Message> get(String conversationId, int lastN) {
        if (lastN <= 0) {
            return List.of();
        }
        List<Conversation> records = conversationMapper.findLatestMessages(
            currentTenantId(), conversationId, lastN);
        
        List<Message> messageList = new ArrayList<>();
        for (Conversation r : records) {
            Message message = switch (r.getType()) {
                case "USER" -> new UserMessage(r.getMessage());
                case "ASSISTANT" -> new AssistantMessage(r.getMessage());
                case "SYSTEM" -> new SystemMessage(r.getMessage());
                default -> null;
            };
            if (message != null) {
                messageList.add(message);
            }
        }
        Collections.reverse(messageList);  // 倒序转正序
        return messageList;
    }

    // 清空会话
    @Override
    public void clear(String conversationId) {
        conversationMapper.delete(new LambdaQueryWrapper<Conversation>()
            .eq(Conversation::getTenantId, currentTenantId())
            .eq(Conversation::getConversationId, conversationId));
    }

    private String currentTenantId() {
        return TenantContext.normalize(MDC.get(TenantContext.TENANT_REQUEST_ATTRIBUTE));
    }
}
```

## ChatClient 配置

### CommonConfiguration

```java
@Configuration
public class CommonConfiguration {

    // 通用聊天客户端
    @Bean
    public ChatClient chatClient(OpenAiChatModel model, ChatMemory chatMemory) {
        return ChatClient.builder(model)
            .defaultSystem("你是一个专业、友好、可靠的AI助手...")
            .defaultAdvisors(
                new SimpleLoggerAdvisor(),                        // 日志记录
                new MessageChatMemoryAdvisor(chatMemory)         // 核心：记忆管理
            )
            .build();
    }

    // 客服客户端（带工具调用）
    @Bean
    public ChatClient serviceChatClient(OpenAiChatModel model,
                                        ChatMemory chatMemory,
                                        CourseTools courseTools) {
        return ChatClient.builder(model)
            .defaultSystem(SystemConstants.CUSTOMER_SERVICE_SYSTEM)
            .defaultTools(courseTools)
            .defaultAdvisors(
                new SimpleLoggerAdvisor(),
                new MessageChatMemoryAdvisor(chatMemory)
            )
            .build();
    }

    // PDF RAG 客户端
    @Bean
    public ChatClient pdfChatClient(OpenAiChatModel model,
                                    ChatMemory chatMemory,
                                    VectorStore vectorStore) {
        return ChatClient.builder(model)
            .defaultSystem("请严格按照上下文的内容进行回答...")
            .defaultAdvisors(
                new SimpleLoggerAdvisor(),
                new MessageChatMemoryAdvisor(chatMemory),
                new QuestionAnswerAdvisor(vectorStore, SearchRequest.builder()
                    .topK(2)
                    .similarityThreshold(0.5)
                    .build())
            )
            .build();
    }
}
```

## 调用流程

### Controller 示例

```java
@RestController
@RequestMapping("/ai")
public class ChatController {

    private final ChatClient chatClient;
    private final ChatHistoryRepository chatHistoryRepository;

    @RequestMapping(value = "/chat", produces = "text/html;charset=utf-8")
    public Flux<String> chat(
            @RequestParam("prompt") String prompt,
            @RequestParam("chatId") String chatId,
            @RequestParam(value = "modelProfile", required = false) String modelProfile) {
        
        // 1. 保存会话记录
        chatHistoryRepository.save("chat", chatId);
        
        // 2. 构建完整的 conversationId
        String conversationId = ConversationIdHelper.build("chat", chatId);
        // 结果: "chat::abc123"
        
        // 3. 调用时传递 conversationId
        return chatClient.prompt()
            .user(prompt)
            .advisors(a -> a.param(
                CHAT_MEMORY_CONVERSATION_ID_KEY,  // Spring AI 常量
                conversationId
            ))
            .stream()
            .content();
    }
}
```

## 技术特性

| 特性 | 实现方式 | 说明 |
|------|---------|------|
| **持久化存储** | MySQL | 重启服务不丢失对话历史 |
| **多租户隔离** | tenant_id 字段 | 不同客户数据完全隔离 |
| **上下文窗口** | 可配置 | `MessageChatMemoryAdvisor` 默认保留最近 20 条 |
| **会话管理** | chatId | 支持按会话查询历史、分页加载 |
| **流式响应** | SSE | 支持流式输出，用户体验更好 |
| **工具调用** | Spring AI Tools | 客服场景支持 Function Calling |

## 与其他方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **本项目 (MySQL + Advisor)** | 持久化、多租户、支持分页查询 | 查询延迟略高 |
| **Redis Session** | 性能高 | 占用内存，不易持久化 |
| **In-Memory** | 速度快 | 重启丢失，难以扩展 |

---

## 总结

本项目的多轮对话核心是 **Spring AI 的 Advisor 模式**。`MessageChatMemoryAdvisor` 是一个拦截器，在每次请求前后自动完成：

- **请求前**：从 MySQL 读取历史消息 → 注入到 prompt
- **响应后**：将本次对话存入 MySQL

开发者只需传递 `conversationId`，其余全部自动化处理。这种设计解耦了对话管理与业务逻辑，便于扩展和维护。
